## 一、需求概述

### 1.1 核心场景

用户一次性选择多条视频，批量发布到多个平台（抖音、快手、微信视频号、西瓜视频、哔哩哔哩），并汇聚各平台数据到统一看板，实现"发出去、看到数据"的闭环。

### 1.2 功能模块

| 模块 | 描述 | 优先级 |
|------|------|--------|
| 平台账号管理 | OAuth 授权绑定、Token 续期、账号状态监控 | P0 |
| 一键分发 | 批量发布、平台特性适配、发布状态实时追踪 | P0 |
| 数据闭环 | 各平台数据拉取（播放/点赞/评论/转发）、统一看板 | P0 |
| 合规检测 | AI 内容合规检测、敏感词/违禁内容检测 | P0 |
| 竞品监测 | 可选，监测竞品账号数据（后期扩展） | P1 |

## 二、平台账号管理

### 2.1 OAuth 授权流程

各平台均采用 OAuth2.0 授权码模式，用户在平台侧授权后，系统存储 `access_token` 与 `refresh_token`。

| 平台 | 授权 endpoint | Token 有效期 | 刷新方式 |
|------|--------------|-------------|---------|
| 抖音 | `https://open.douyin.com/oauth/client_token` | 24h | refresh_token 30天 |
| 快手 | `https://open.kuaishou.com/oauth2/access_token` | 7天 | refresh_token 30天 |
| 视频号 | 微信公众平台 OAuth | 2h | 通过 authorizer_access_token 续期 |
| 西瓜 | 头条开放平台 OAuth | 24h | refresh_token |
| 哔哩哔哩 | `https://open.bilibili.com/oauth2/authorize` | 30天 | refresh_token |

### 2.2 数据模型

```python
# 新建：app/models/platform_account.py

class PlatformAccount(SQLModel, table=True):
    """平台账号绑定表"""
    __tablename__ = "platform_accounts"

    id: str = Field(default=None, primary_key=True, max_length=12)  # 如 PAC_000001
    user_id: str = Field(foreign_key="users.id")
    platform: Platform                          # 平台枚举
    platform_user_id: str = ""                  # 平台侧用户ID
    platform_user_name: str = ""                 # 平台昵称
    platform_avatar: str = ""                    # 平台头像
    access_token: str = ""                       # 访问令牌（加密存储）
    refresh_token: str = ""                      # 刷新令牌（加密存储）
    token_expires_at: Optional[datetime] = None # Token 过期时间
    account_status: AccountStatus = Field(default=AccountStatus.ACTIVE)  # 账号状态
    extra_data: str = "{}"                      # 平台特有扩展数据（JSON）
    created_at: datetime = Field(default_factory=datetime.utcnow)
    updated_at: datetime = Field(default_factory=datetime.utcnow)


class AccountStatus(str, Enum):
    ACTIVE = "active"         # 正常
    TOKEN_EXPIRED = "expired" # Token 过期需续期
    AUTH_REVOKED = "revoked"  # 用户撤销授权
    DISABLED = "disabled"     # 被禁用
```

### 2.3 Token 续期机制

- **后台定时任务**：每天凌晨扫描 `token_expires_at` 在 7 天内的账号，自动用 `refresh_token` 续期
- **发布前置检查**：发布前检查 token 状态，若已过期先续期再发布
- **续期失败处理**：标记为 `TOKEN_EXPIRED`，通知用户重新授权

### 2.4 API 设计

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/v1/platforms` | 获取支持的平台列表 |
| GET | `/api/v1/accounts` | 获取当前用户绑定的账号列表 |
| GET | `/api/v1/accounts/{platform}/auth_url` | 获取平台 OAuth 授权 URL |
| POST | `/api/v1/accounts/callback` | OAuth 回调，交换 Token |
| DELETE | `/api/v1/accounts/{account_id}` | 解绑账号 |
| POST | `/api/v1/accounts/{account_id}/refresh` | 手动刷新 Token |
| GET | `/api/v1/accounts/{account_id}/status` | 查询账号状态（Token 有效期） |

## 三、一键分发

### 3.1 批量发布流程

```
用户选择 N 条视频 × M 个平台
        ↓
创建 N×M 条 PublishRecord（状态：PENDING）
        ↓
对每个平台账号，检查 Token 有效性
        ↓
按平台分组，并发发布（每个平台独立异步任务）
        ↓
实时推送 WebSocket 状态更新
        ↓
发布完成，更新状态（PUBLISHED / FAILED）
```

### 3.2 平台特性适配

不同平台对视频参数有不同要求，发布时需做适配：

| 平台 | 封面 | 标题字数 | 标签格式 | 话题格式 |
|------|------|---------|---------|---------|
| 抖音 | 必须，jpg/png，<5MB | ≤30字 | `#标签` 插入正文 | `#话题1 #话题2` |
| 快手 | 必须，jpg/png，<5MB | ≤40字 | `标签1 标签2` | 话题用正文 |
| 视频号 | 必须，jpg/png，<5MB | ≤100字 | `#话题` | `#话题1;`（分号分隔） |
| 西瓜 | 必须，jpg/png | ≤50字 | `#标签` | 同抖音 |
| 哔哩哔哩 | 必须，jpg/png | ≤80字 | `$标签$` | 分区选择，必选 |

**适配策略**：
- 前端配置每条视频的平台差异化字段（标题/封面/标签）
- 后端存储时统一结构，发布时按平台模板转换
- 未配置的平台用默认值（原始标题、默认封面）

### 3.3 发布请求数据结构

```python
class BatchPublishRequest(SQLModel):
    video_ids: list[str]              # 要发布的视频 ID 列表
    platform_accounts: list[str]      # 目标平台账号 ID 列表
    platform_overrides: dict[str, PlatformPublishConfig] = {}  # 平台差异化配置
    # PlatformPublishConfig: { title, cover_id, tags, categories }


class PublishTask(SQLModel):
    """发布任务（对应一条 PublishRecord）"""
    id: str
    video_id: str
    platform: Platform
    account_id: str
    status: PublishStatus
    platform_video_id: str = ""
    platform_url: str = ""
    error_message: str = ""
    retry_count: int = 0
    created_at: datetime
    updated_at: datetime
```

### 3.4 实时状态推送

使用 WebSocket 推送发布状态：

```python
# WebSocket 频道: /ws/publish/{user_id}
{
    "type": "publish_progress",
    "task_id": "PUB_000001",
    "status": "publishing",       # pending/publishing/published/failed
    "progress": 0.6,              # 整体进度（已发布数/总数）
    "platform": "douyin",
    "message": "视频上传中...",
    "timestamp": "2026-04-03T12:00:00Z"
}
```

### 3.5 失败重试机制

- 单条失败自动重试 3 次，间隔 30s / 60s / 300s（指数退避）
- 永久失败记录 `error_message`，用户可手动重试
- 平台级限流（429）自动降速等待后重试

## 四、数据闭环

### 4.1 数据拉取架构

```
定时任务（每小时）
        ↓
遍历所有已发布且在有效期内的 PublishRecord
        ↓
按平台分组，调用各平台数据 API
        ↓
解析并存储到 publish_stats 表
        ↓
更新聚合数据缓存
```

### 4.2 数据模型

```python
class PublishStats(SQLModel, table=True):
    """发布统计数据表"""
    __tablename__ = "publish_stats"

    id: str = Field(default=None, primary_key=True, max_length=12)
    publish_record_id: str = Field(foreign_key="publish_records.id")
    platform: Platform
    views: int = Field(default=0)           # 播放量
    likes: int = Field(default=0)           # 点赞数
    comments: int = Field(default=0)        # 评论数
    shares: int = Field(default=0)          # 转发/分享数
    followers: int = Field(default=0)       # 涨粉数（平台提供时）
    platform_updated_at: Optional[datetime] = None  # 平台侧数据更新时间
    created_at: datetime = Field(default_factory=datetime.utcnow)
    updated_at: datetime = Field(default_factory=datetime.utcnow)
```

### 4.3 各平台数据 API

| 平台 | API | 数据维度 | 频率限制 |
|------|-----|---------|---------|
| 抖音 | `https://open.douyin.com/data/extern/user/item/list` | 播放/点赞/评论/转发 | 100次/分钟 |
| 快手 | `https://open.kuaishou.com/openapi/feed/stats` | 播放/点赞/评论 | 1000次/天 |
| 视频号 | 微信数据分析接口 | 播放/点赞/评论/转发 | 有限制 |
| 西瓜 | 头条数据接口 | 播放/点赞/评论 | 频率限制 |
| 哔哩哔哩 | `https://api.bilibili.com/x/stat` | 播放/点赞/评论 | 无明确限制 |

| 方法 | 路径 | 描述 |

## 五、合规检测

### 5.1 检测维度

| 检测项 | 实现方式 | 阈值 |
|--------|---------|------|
| 音乐/BGM 版权 | 音频指纹比对（AfMusic / ACRCloud） | 版权库无匹配才通过 |
| 画面侵权 | 视频指纹 + 截图比对（百度/腾讯视觉） | 相似度 > 0.85 报警 |
| 敏感人物 | 人脸检测 + 黑名单库 | 任意匹配即报警 |
| 敏感词 | 本地词库 + 大模型检测 | 命中即报警 |
| 违禁内容 | 违规内容识别（图片/音频/文字） | 违规即报警 |
| 平台规则违禁 | 各平台社区规范检测 | 按平台规则判定 |

### 5.2 合规检测 API

```python
class ComplianceCheckRequest(SQLModel):
    video_id: str
    platforms: list[Platform]   # 指定检测哪几个平台的规则
    check_music: bool = True
    check_face: bool = True
    check_sensitive: bool = True


class ComplianceCheckResult(SQLModel):
    passed: bool
    overall_risk_level: RiskLevel  # LOW/MEDIUM/HIGH/BLOCK
    issues: list[ComplianceIssue]
    platform_results: dict[str, PlatformComplianceResult]


class ComplianceIssue(SQLModel):
    type: str            # "music_copyright" / "face_match" / "sensitive_word" / "forbidden_content"
    severity: str        # "warning" / "danger"
    description: str
    evidence: str        # 证据描述，如"匹配音乐：XXX，相似度 92%"
    suggestion: str      # 修复建议
```

### 5.3 检测时机

- **生成后发布前**：视频渲染完成、用户发起发布前，自动触发合规检测
- **强制拦截**：高风险（BLOCK）直接拦截发布，提示用户修改
- **警告通过**：中风险（MEDIUM/HIGH）可让用户确认后继续发布

### 5.4 实施建议

| 检测项 | 推荐方案 | 成本 |
|--------|---------|------|
| 敏感词 | `apeishu/tencentcloud` 文本审核 API | 低 |
| 图片鉴黄/暴恐 | 阿里云内容安全 / 腾讯云内容安全 | 按次计费 |
| 音频版权 | ACRCloud / 腾讯音乐审核 | 按次计费 |
| 人脸识别 | 百度人脸识别 / 腾讯人脸比对 | 按次计费 |

### 6.3 核心扩展点

```python
class BaseOAuth(ABC):
    platform: Platform

    @abstractmethod
    def get_authorization_url(self, state: str) -> str: ...

    @abstractmethod
    async def exchange_code(self, code: str) -> dict: ...

    @abstractmethod
    async def refresh_token(self, refresh_token: str) -> dict: ...
```

class BasePublisher(ABC):

## 七、MVP 范围定义

### 7.1 MVP 目标

**MVP 完成标准**：用户可绑定抖音/快手账号，将生成的多条视频一键发布到任一平台，并查看发布状态和数据统计。

### 7.2 MVP 功能清单

#### 账号管理（MVP）

- [ ] `PlatformAccount` 数据模型
- [ ] 账号列表查询 API
- [ ] 抖音 OAuth 绑定流程（授权 URL 生成 → 回调 → Token 存储）
- [ ] 快手 OAuth 绑定流程
- [ ] Token 续期逻辑（手动触发 + 定时任务）
- [ ] 账号解绑

#### 一键分发（MVP）

- [ ] 批量发布请求 API（支持多视频 × 单/多平台）
- [ ] 平台差异化配置（标题、封面、标签）
- [ ] 后台发布任务（`publish_task` 扩展，支持批量）
- [ ] 发布状态 WebSocket 推送
- [ ] 发布失败重试机制
- [ ] 发布历史记录查询

#### 数据闭环（MVP）

- [ ] `PublishStats` 数据模型
- [ ] 抖音数据拉取服务（播放/点赞/评论）
- [ ] 快手数据拉取服务（播放/点赞/评论）
- [ ] 定时数据同步任务（每小时）
- [ ] 数据看板 API（扩展现有 `/analytics`）
- [ ] 单条视频数据详情 API

#### 合规检测（MVP）

- [ ] `ComplianceCheckResult` 数据模型
- [ ] 文本敏感词检测服务（本地词库 + 大模型）
- [ ] 视频发布前自动触发检测
- [ ] 检测结果展示（前端展示警告/拦截）

### 7.3 MVP 不包含

以下功能放在 P1 迭代，不在 MVP 范围：

- 微信视频号、西瓜视频、哔哩哔哩对接
- 音乐版权检测（需 ACRCloud 等第三方服务）
- 人脸识别（需百度/腾讯人脸服务）
- 封面自动生成建议
- 竞品监测
- 定时发布（`scheduled_at` 字段已存在，但 MVP 只做即时发布）
- 发布草稿箱

### 7.4 MVP 里程碑

| 阶段 | 内容 | 预计工时 |
|------|------|---------|
| M1 | 数据模型 + OAuth 基类 + 抖音/快手 OAuth | 2天 |
| M2 | 批量发布 API + 后台任务 + WebSocket | 2天 |

## 八、平台申请指南

在正式对接前，需提前在各平台注册开发者账号并申请权限：

### 8.1 抖音

1. 前往 [抖音开放平台](https://open.douyin.com/)
2. 注册开发者账号，创建应用
3. 申请权限：`video.upload`（上传）、`video.publish`（发布）、`data.external.user.item.list`（数据）
4. 配置 OAuth 回调地址
5. 获取 `Client Key` 和 `Client Secret`

### 8.2 快手

1. 前往 [快手开放平台](https://open.kuaishou.com/)
2. 注册开发者账号，创建应用
3. 申请权限：视频上传、视频发布、数据统计
4. 获取 `App ID` 和 `App Secret`

### 8.3 视频号（微信）

1. 在微信公众平台注册视频号
2. 关联微信开放平台账号
3. 申请数据分析能力（需企业认证）

| 平台 API 变更 | 各平台 API 经常升级，可能导致对接失效 | 抽象适配层，API 变更只改实现不换接口 |
| 平台限流 | 发布/数据拉取被平台限流 | 实现指数退避 + 任务队列 |
| Token 过期未续期 | 发布时发现 Token 失效 | 定时扫描 + 发布前预检 |
| 合规误判 | 误杀正常内容影响用户体验 | 设置白名单 + 用户申诉通道 |
| 第三方服务成本 | 合规检测 API 按次计费，成本不可控 | MVP 先用免费额度，设置每日限额 |
